🚀 БЫСТРЫЙ СТАРТ

1. Установить зависимости:
   npm install

2. Запустить в режиме разработки:
   npm run dev

3. Сервер слушает на: http://localhost:3001

4. CORS настроен для фронтенда: http://localhost:5173

🔌 API ENDPOINTS (6 основных маршрутов)

/api/users              - Пользователи
/api/auth/login         - Аутентификация
/api/materials          - Материалы (CRUD + receive/consume)
/api/techcards          - Технологические карты
/api/tasks              - Производственные задачи
/api/dashboard          - Статистика
/api/events             - События системы

Подробнее в API_EXAMPLES.md (20+ примеров curl)

💾 DATABASE SCHEMA (6 таблиц)

users                   - Пользователи системы (5 ролей)
materials               - Материалы на складе (с batch tracking)
tech_cards              - Технологические карты (рецепты)
tech_card_items         - Связь материалов и рецептов
production_tasks        - Производственные задачи (с tracking)
events                  - Логирование действий

Все таблицы созданы автоматически при первом запуске.

⚙️ ТЕХНИЧЕСКИЕ ОСОБЕННОСТИ

✅ SQLite с WAL режимом (отличная параллелизация)
✅ Foreign Keys включены (целостность данных)
✅ Синхронный драйвер (no callback hell)
✅ Параметризованные запросы (защита от SQL injection)
✅ HTTP логирование (morgan)
✅ CORS настроен
✅ Global error handler
✅ Демо-данные (9 материалов, 4 тех. карты, 6 задач)

🔑 КРИТИЧЕСКАЯ ЛОГИКА

1. Создание задачи:
   - Привязка к технологической карте
   - Назначение оператору
   - Планирование количества

2. Завершение задачи:
   - Фиксация произведенного количества
   - АВТОМАТИЧЕСКОЕ СПИСАНИЕ материалов со склада
     (на основе рецепта и количества)
   - Запись события в систему

3. Статистика:
   - Расчет процента дефектов в реальном времени
   - Алерты о низких остатках материалов
   - Ленты событий с JOIN пользователей
