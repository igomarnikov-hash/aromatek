ШАГИ УСТАНОВКИ И ЗАПУСКА

1️⃣  УСТАНОВИТЬ ЗАВИСИМОСТИ

   cd /Users/admin/Documents/Claude/Projects/SaaS/production-system/backend
   npm install

   ⏱️  Время: 1-2 минуты (первый раз)

2️⃣  ЗАПУСТИТЬ СЕРВЕР

   npm run dev

   Вывод должен быть примерно такой:
   ┌────────────────────────────────────────────────────────────┐
   │ АромаПро API запущен на http://localhost:3001              │
   │ База данных инициализирована с демо-данными               │
   └────────────────────────────────────────────────────────────┘

3️⃣  ПРОВЕРИТЬ ЧТО РАБОТАЕТ

   В другом терминале выполнить:

   curl http://localhost:3001/api/users

   Должно вернуться:
   {
     "success": true,
     "data": [
       {"id": 1, "name": "Александр Петров", "role": "admin", ...},
       ...
     ]
   }

4️⃣  ЗАПУСТИТЬ ФРОНТЕНД (опционально)

   В отдельном терминале:
   cd /Users/admin/Documents/Claude/Projects/SaaS/production-system/frontend
   npm install && npm run dev

   Фронтенд будет на: http://localhost:5173

5️⃣  ОТПРАВИТЬ ТЕСТОВЫЙ ЗАПРОС

   curl -X POST http://localhost:3001/api/auth/login \
     -H "Content-Type: application/json" \
     -d '{"user_id": 1}'

   Ответ:
   {
     "success": true,
     "data": {
       "id": 1,
       "name": "Александр Петров",
       "role": "admin",
       "avatar_color": "#2563eb"
     }
   }

❓ РЕШЕНИЕ ПРОБЛЕМ

Проблема: "Cannot find module 'express'"
Решение: npm install

Проблема: "Port 3001 already in use"
Решение: PORT=3002 npm run dev
         или: lsof -i :3001 && kill -9 <PID>

Проблема: "EACCES permission denied"
Решение: chmod -R 755 . && npm install

Проблема: "Database is locked"
Решение: Удалить production.db* файлы и перезагрузить сервер
