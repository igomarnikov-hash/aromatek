# Система управления производством - Фронтенд

Полнофункциональный React-фронтенд для системы управления производством воздухоароматизаторов "АромаПро".

## Структура проекта

```
frontend/
├── index.html              # Главный HTML файл
├── package.json            # Зависимости и скрипты
├── vite.config.js         # Конфигурация Vite
├── src/
│   ├── main.jsx           # Точка входа приложения
│   ├── App.jsx            # Главный компонент приложения
│   ├── api.js             # API клиент
│   ├── styles/
│   │   └── index.css      # Глобальные стили (промышленный дизайн)
│   ├── context/
│   │   └── AuthContext.jsx # Контекст аутентификации
│   ├── components/
│   │   ├── Layout.jsx     # Главная раскладка приложения
│   │   └── Sidebar.jsx    # Боковая панель навигации
│   └── pages/
│       ├── LoginPage.jsx  # Страница входа
│       ├── DashboardPage.jsx # Дашборд
│       ├── MaterialsPage.jsx # Управление материалами/складом
│       ├── TechCardsPage.jsx # Технологические карты
│       ├── TasksPage.jsx  # Производственные задачи
│       ├── WorkshopPage.jsx # Интерфейс оператора цеха
│       └── SettingsPage.jsx # Настройки системы
└── .gitignore
```

## Установка и запуск

### Шаг 1: Установка зависимостей

```bash
cd frontend
npm install
```

### Шаг 2: Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно на `http://localhost:5173`

### Шаг 3: Сборка для продакшена

```bash
npm run build
```

Собранные файлы появятся в папке `dist/`

### Шаг 4: Preview собранного приложения

```bash
npm run preview
```

## Требования

- Node.js 16+ и npm
- Backend API должен работать на `http://localhost:3001`

## Архитектура

### Аутентификация

- Использует контекст React для управления состоянием пользователя
- Данные пользователя хранятся в sessionStorage
- Поддерживает роли: Владелец, Технолог, Нач. производства, Кладовщик, Оператор

### API интеграция

- Класс `api.js` предоставляет методы: `get()`, `post()`, `put()`
- Автоматическая обработка ошибок
- Поддержка CORS через Vite прокси

### Стили

- CSS переменные для цветов и размеров
- Минималистичный промышленный дизайн
- Темная тема (#0f172a основной фон)
- Цветовая кодировка: зеленый=норма, желтый=предупреждение, красный=критично, синий=информация
- Полностью адаптивен (мобильный, планшет, десктоп)

## Страницы приложения

### Дашборд (/)
- Общая статистика: производство, задачи, события
- Статус материалов и задач с прогресс-барами
- Таблица последней активности

### Склад (/materials)
- Список материалов с уровнями запасов
- Система оповещений (критический уровень, предупреждения)
- Функции: добавление материала, поступление, расход

### Техкарты (/techcards)
- Управление технологическими картами
- CRUD операции
- Отображение материалов и этапов производства

### Задачи (/tasks)
- Создание и управление производственными задачами
- Назначение исполнителей
- Отслеживание прогресса (начать, завершить, отметить дефект)
- Фильтрация по статусу и приоритету

### Цех (/workshop)
- Специализированный интерфейс для операторов
- Текущая активная задача с таймером
- Технологическая карта для текущей задачи
- Очередь ожидающих задач
- Завершенные задачи

### Настройки (/settings)
- Основные параметры компании
- Управление пользователями (для владельца)
- Параметры уведомлений
- Параметры безопасности и резервного копирования

## Особенности дизайна

### Промышленный стиль
- Высокий контраст для использования в цеховых условиях
- Большие интерактивные элементы
- Четкая визуальная иерархия
- Цветовая кодировка статусов

### Гибкая боковая панель
- Развернутый режим: 250px (показывает названия)
- Свернутый режим: 64px (только иконки)
- Адаптивная: на мобильных - скрывается (overlay)
- Быстрые переходы между разделами

### Отзывчивость
- Мобильные устройства (480px): компактная версия
- Планшеты (768px): оптимизированная раскладка
- Десктоп (1200px+): полный функционал

## API эндпоинты

Приложение работает со следующими эндпоинтами:

- `GET /api/users` - получить список пользователей
- `POST /api/auth/login` - вход в систему
- `GET /api/dashboard` - данные дашборда
- `GET /api/materials` - список материалов
- `POST /api/materials` - добавить материал
- `PUT /api/materials/:id` - обновить материал
- `POST /api/materials/:id/receive` - поступление материала
- `POST /api/materials/:id/consume` - расход материала
- `GET /api/materials/alerts` - оповещения материалов
- `GET /api/techcards` - список техкарт
- `POST /api/techcards` - создать техкарту
- `PUT /api/techcards/:id` - обновить техкарту
- `GET /api/tasks` - список задач
- `POST /api/tasks` - создать задачу
- `PUT /api/tasks/:id` - обновить задачу
- `POST /api/tasks/:id/start` - начать задачу
- `POST /api/tasks/:id/complete` - завершить задачу
- `POST /api/tasks/:id/defect` - отметить дефект
- `GET /api/events` - получить события

## Демо-данные

В случае недоступности API приложение загружает встроенные демо-данные:
- 5 пользователей с разными ролями
- 5 материалов со статусами
- 3 технологические карты
- 5 производственных задач

## Разработка

### Добавление новой страницы

1. Создать компонент в `src/pages/NewPage.jsx`
2. Импортировать в `App.jsx`
3. Добавить маршрут в Routes
4. (Опционально) Добавить пункт меню в Sidebar.jsx

### Добавление нового API вызова

```javascript
// В component
import api from '../api';

const data = await api.get('/api/endpoint');
```

### Изменение стилей

Все стили в `src/styles/index.css`. Используйте CSS переменные:
- `--primary` для основного цвета
- `--success`, `--warning`, `--danger`, `--info` для статусов
- `--bg-primary`, `--bg-secondary` для фонов
- `--text-primary`, `--text-secondary` для текста
